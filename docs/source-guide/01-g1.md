# 01 · 内核与启动骨架

> backend/app · kernel / 入口 / lib / jobs

一次 chat turn 由 **main.lifespan** 装配环境后启动；**TurnRunner**(runner.py) 持有该 turn 并向 SSE 订阅者扇出/重放。turn 主循环在固定点调 **pipeline.run_stage**（钩子），每个 iter 通过 **llm_loop.stream_one_iter**(配 retry) 做一次流式 LLM 调用，回调经 **sse.py** 打包（类型契约见 sse_protocol）推给前端；工具调用经 tools/registry 门面落到 **kernel/tool_registry**，按 tool_protocol 契约 dispatch。

!!! note "建议阅读顺序"
    config → trace → main → tool_protocol → tool_registry → llm_loop → retry → pipeline → event_bus → runner → sse → sse_protocol → errors → jobs

**优先级：** 🔴 核心　·　集中式 Pydantic Settings：全部运行时配置的单一来源。

- `Settings(BaseSettings)` 读 .env.local：DEEPSEEK_*、SHELL 白名单、SANDBOX 上限、MAX_LOOPS/MAX_SUB_LOOPS、STATE_BACKEND/REDIS/POD、AUTH、STRIP_OLD_REASONING
- `BACKEND_ROOT`/`PROJECT_ROOT` 锚定路径；validator 解析 CORS/归一化 search provider
- 模块级单例 `settings = Settings()` 全工程 import

**依赖：** pydantic-settings；被全工程广泛引用

### `backend/app/main.py`

**优先级：** 🔴 核心　·　FastAPI 入口：启动/关闭生命周期、状态后端装配、引导恢复、全路由挂载。

### `backend/app/kernel/llm_loop.py`

**优先级：** 🔴 核心　·　主/子共享的『单次 LLM 流式 iter』内核。

- `StreamIterResult`：累积 content/reasoning/tool_calls/finish_reason/usage + 取消标记
- `stream_one_iter()`：一次 stream=True 调用，逐 chunk 累积并经 on_content_delta/on_tool_* 回调推 SSE；cancel_event 即中断
- `_strip_old_reasoning()` 用占位符替换历史 reasoning 保 KV 前缀；`stream_with_retry()` 瞬时错误退避重试

**依赖：** config、lib.retry.is_transient；编排留在 chat/sub_agent

**优先级：** 🔴 核心　·　进程内单 turn 运行器：让 turn 在 HTTP 请求结束后存活并向多订阅者扇出+断线重放。
