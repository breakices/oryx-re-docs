# 02 · 数据层：领域实体 + 持久化

> backend/app · domain / db

刻意分两层：**domain/** 是纯 Python(仅 dataclass)，承载字段+不变量方法+状态机(Turn pending→running→done/failed/cancelled、BgTask 多一个 waiting 态)；**db/** 持 SQLAlchemy 2.0 ORM(models.py) 与按域拆分的 repo；**db/mapping.py** 做 ORM↔domain 双向翻译。这是渐进式迁移：想要不变量保护的调用方才 to_*→调方法→apply_*→repo.update。

!!! note "建议阅读顺序"
    domain/__init__ → session → turn → bg_task → message → memory → workspace → db/models → mapping → engine → repo/_base → repo/sessions → turns → messages → pending → bg → memories → workspaces → users

**依赖：** orm.Session；mapping.to/apply_session
