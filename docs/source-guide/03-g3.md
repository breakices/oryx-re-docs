# 03 · 横切关注 + 基础设施

> backend/app · capabilities / infra

**infra/** 每个子系统 = 一个 Protocol/ABC + 若干 adapter + 一个单例选择器：state(StateStore 六子协议 / InProcess vs Redis 决定单机还是多 pod)、sandbox、trace_sink(File 本地 vs ClickHouse 生产)、websearch(Tavily+Volcengine 经 ChainProvider 回落)、llm(main/light/vision 三通道)。**capabilities/** 用 ContextVar 把身份/LLM 路由隐式贯穿整个 asyncio 任务树，子 agent 与后台回合无需透传即可继承。

!!! note "建议阅读顺序"
    state/protocols → singleton → in_process → redis → auth_context → permission → audit → llm/deepseek → light → vision → sandbox/local_docker → trace_sink(base/file/clickhouse) → websearch(base/tavily/volcengine/chain/factory) → pod_routing

### `backend/app/infra/state/protocols.py`

**优先级：** 🔴 核心　·　StateStore 总协议 + 六子协议，全仓最能体现 Protocol+adapter。

- `PubSubBus`(按 session 扇出) / `SignalHub`(命名 ack 信号 emit/wait/terminal) / `WakeHub`(遇活动提前返回)
- `LockManager` / `KVStore`(带 TTL) / `StreamLog`(有界环形，审计)
- StateStore 聚为 pubsub/signals/wakes/locks/kv/audit

**依赖：** Protocol、asyncio

### `backend/app/capabilities/auth_context.py`

**优先级：** 🔴 核心　·　请求范围鉴权上下文，ContextVar 镜像让深层调用读身份。

**依赖：** config、db.repo、impersonation.service

### `backend/app/infra/pod_routing.py`

**优先级：** 🟡 重要　·　多 pod 属主路由：转发/重定向到拥有资源的 pod，HMAC 防伪。

- HMAC 签名 + validate_routed_request(时钟偏移+compare_digest)
- `owner_for_request_path` 仅从 URL 解析属主；`proxy_to_owner`(307 重定向 / aiohttp 流式转发)
- 全由 POD_MULTI_ENABLED 门控，单机走旧直连

**依赖：** aiohttp、hmac、config、db.repo
