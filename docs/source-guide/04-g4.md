# 04 · 核心对话编排（应用心脏）

> backend/app/features · chat / compact

应用的心脏。router 是 HTTP/SSE 边缘（/chat 立即返回 ack、/stream 流式回放），**turn.py** 是流式 agent 主循环（多轮 LLM 迭代、工具分发、deferred wait、pending-A 注入、HARD 内联 compact、崩溃恢复）。**constants.py** 的 SYSTEM_PROMPT/COMPACT_PROMPT 是必须逐字节稳定的 **KV 缓存锚点**。compact 把较早历史折叠成结构化摘要 + restore 种子。

!!! note "建议阅读顺序"
    constants → schemas → router → messages → bootstrap → turn → deferred → lifecycle → compact/__init__ → auto_extract → digest → tool_presentation → replay → chat/__init__

---

### `backend/app/features/chat/constants.py`

**优先级：** 🔴 核心　·　chat 共享字面量：主 agent SYSTEM_PROMPT、COMPACT_PROMPT、各阈值。

- `SYSTEM_PROMPT` 是 **KV 缓存前缀锚点**(必须 byte-stable)：定义 workspace 路径规则、read/edit/write/shell/run_sandbox 语义、web/math/skills/sub-agent/narration 行为
- `COMPACT_PROMPT`：强指令覆盖，产出结构化中文摘要骨架 + ≤8 个仅限 read_file/load_memory 的 restore 种子
- 阈值 HARD=900k/SOFT=800k、MAX_RESTORE_SEEDS=8、MAX_PENDING=3、REASONING_COMPACTED 占位符

**依赖：** config.settings

### `backend/app/features/chat/router.py`

**优先级：** 🔴 核心　·　chat HTTP/SSE 边缘：请求解析、鉴权、远程 pod 代理、委派。

- `POST /chat`：rehydrate→gate_and_preload(并行 lite 闸门)→持久化 user+create_turn→注册 TurnRunner→create_task(run_turn)→立即返 ChatAck
- `GET /turns/{tid}/stream`：有活 runner 则 subscribe() 逐 chunk yield，过期则 replay_from_db
- cancel / pending CRUD(A/B 两态) / rewind / 手动 compact

**依赖：** runner、repo、chat 聚合、compact、sse_starlette、pod_routing

### `backend/app/features/chat/turn.py`

**优先级：** 🔴 核心　·　主 turn 编排：流式 agent 主循环（全应用最重要文件）。

- `run_turn`：装配 messages(head + 过滤历史 + pre-inject cycles + triage/spawn hint)，进入 while loop_idx<MAX_LOOPS
- 每轮：pop_pending_a 注入中途指令 → build_api_messages 处理 deferred 尾包 → stream_one_iter(3 次重试) → 工具 dispatch/emit
- **原子提交** assistant+sync tool 行于一次 bulk_append(崩溃前 DB 不变便 resume)；HARD compact 内联折叠
- 结束 terminate_turn→chain_pending_b→钩子→写 TraceRecord；`run_refusal`(安全短路)、`resume_turn`(崩溃恢复)

**依赖：** runner、repo、kernel.llm_loop/pipeline、bootstrap、compact、messages、deferred

### `backend/app/features/chat/messages.py`

**优先级：** 🔴 核心　·　消息整形：构建 OpenAI messages 数组、DB↔OpenAI 互转、wire 裁剪。

- `build_head_messages`：缓存锚头 = 季度时间窗 + SYSTEM_PROMPT；细时间戳放每消息体不污染锚
- `build_api_messages`：deferred wait 结果用合成尾包，**绝不重排已有位置**保 KV 前缀
- wire 侧：wire_tool_chunk(按 DEV_MODE 剥 params/result)、result_for_view(read_file 截 1200 字)

**依赖：** constants、tool_presentation、tool_registry

### `backend/app/features/chat/bootstrap.py`

**优先级：** 🔴 核心　·　会话入口编排：lite 闸门并行预加载 + rehydrate + pending-B 链接。

- `gate_and_preload`：gather 跑 triage+候选记忆+skill gate+spawn advisor，去重已加载保前缀，打包 turn_meta
- `maybe_rehydrate`：找最近 compact_summary，未执行则 dispatch 其 restore 种子并标 seeds_executed
- `chain_pending_b`：turn 成功后以 B[0] 作新 user spawn 新 turn

**依赖：** repo、memory_ranker、skill_gate、triage_guard、pre_inject

### `backend/app/features/compact/__init__.py`

**优先级：** 🔴 核心　·　compact 主体：token 度量、cutoff、SOFT/HARD 内联 compact、compact turn。

- 度量/cutoff：session_context_tokens(阈值依据)、pick_compact_cutoff、estimate_saved
- `compact_inline`：发 compact_start/done，新建 compact turn+runner 后 run_compact 再 rehydrate
- `run_compact`：**单次 LLM 调用**(同 SYSTEM_PROMPT 保前缀 + 尾追 COMPACT_PROMPT)，产摘要+种子，持久化 compact_summary/compact_boundary，被折叠行打 compacted_into，种子不在此执行

**依赖：** repo、llm_loop、chat.constants/messages、auto_extract
