# 05 · Agent 派生与工具执行

> backend/app/features · sub_agent / sandbox_runner / shell_tool / web_tool / next_options / pre_inject

**sub_agent** 派生隔离子会话跑受限工具白名单的 LLM 循环，全部活动经 bg_events+hub 流回父会话（SUB_SYSTEM_PROMPT 是 KV 锚）。**sandbox_runner** 是『无 LLM 子智能体』：docker network=none + cap-drop 跑代码，复用同套 emit/signal 通道。**shell_tool** 有九层静态护栏 + lite 恶意检测。**next_options** 是 FormTool 范例，**pre_inject** 用『假 tool_call 持久化』保 KV 前缀。

!!! note "建议阅读顺序"
    presets → sub_agent/runner → hub → emit → service → tool → schemas/router → sandbox(runner/service/tool) → shell(lite/tool) → web_tool → pre_inject → next_options

---

### `backend/app/features/sub_agent/service.py`

**优先级：** 🔴 核心　·　子智能体核心编排：派生 sub-Session + 受限工具 LLM 循环 + 流式广播。

- `SUB_SYSTEM_PROMPT` 固定首条 system(KV 锚)；finance preset 再追加
- `spawn()` 建 sub-Session+BgTask+turn，注册 SubAgentRunner，create_task(_run_sub_turn) 立即返回 bg
- `_run_sub_turn`：stream_one_iter + preset 白名单 dispatch + 每 iter 原子提交；MAX_SUB_LOOPS + 运行时看门狗
- 终止靠 emit(terminal=True)；内容只经 deliver_emit→TurnPendingInput 进父上下文，signal 仅 ack

**依赖：** emit/hub/presets/runner、llm_loop、deepseek、repo

### `backend/app/features/sub_agent/emit.py`

**优先级：** 🔴 核心　·　bg_task 统一 emit 投递路径（LLM 子智能体与 docker 任务共用）。

- progress(terminal=False)只发 bg_event+signal(不进父上下文控 token)；terminal 才写 TurnPendingInput+wake+latch report
- **不变量**：pending_input 必须先于 emit_signal 提交，否则 parked waiter race pop 漏读
- MAX_INJECT_CONTENT_CHARS=20000 截断；scrub_nulls 清 NUL

**依赖：** hub、repo、惰性 import chat.maybe_spawn_digest_turn

### `backend/app/features/sandbox_runner/service.py`

**优先级：** 🔴 核心　·　process bg_task 执行器：『无 LLM 子智能体』，执行体是 docker 容器。

- `spawn_sandbox`：_clamp_resources 到 settings 上限，校验 SANDBOX_MAX_CONCURRENT
- `_run_process` 安全：source_dir 必须**严格位于** workspace root 之下(拒整盘挂载)；输出落 outputs/<bg_id>/
- stdout 扫 `__ORYX__ {json}` → deliver_emit；普通行只 silent log；三路 wait(exit/cancel/timeout)

**依赖：** sub_agent.emit/hub、infra.sandbox.local_docker、workspace.service、repo

### `backend/app/features/shell_tool/tool.py + lite.py`

**优先级：** 🔴 核心　·　会话沙箱化 workspace 内跑 shell，九层护栏 + lite 恶意检测 + 输出脱敏。

- 九层(run_shell 顺序)：①路径归一②拒 .. 越界③拒 $()/`` 元编程④拒裸 $VAR⑤拒 find -exec/-delete⑥拒宿主路径⑦ls→list_dir⑧cat→read_file⑨二进制白名单+lite is_malicious_shell(fail-closed)
- 引号感知解析；执行用 bash -c + **干净 env**(不继承 os.environ，钉死 workspace 安全值)
- 输出 _hide_real_root + scrub_nulls + lite summary

**依赖：** settings、repo、shell_tool.lite、workspace.service
