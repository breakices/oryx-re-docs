# 07 · 金融子系统

> backend/app/features · finance / finance_data / finance_memory / finance_portfolio

主线：稳定金融工具(finance_quote/news/search_data/statement/provider_status) → provider 适配器 + auth 解析器(oauth/sdk_session/static)模式 → ProviderResult/registry 统一结果与 fallback → finance_memory(以 Oryx DB 为权威，SQL 图上下文 + records/entities/edges/facts/gaps) → finance_portfolio(纸面研究，复用行情做回测)。凭据与原始响应都隐藏在稳定契约层之后。

!!! note "建议阅读顺序"
    finance_data/schemas → auth(base/static/oauth/sdk_session) → providers/base → registry → yahoo → 其余 provider → finance_data/service → tool → finance(schemas/service/router) → finance_memory(schemas→service→jobs→tool→router) → finance_portfolio(schemas→service→tool→router)

### `backend/app/features/finance_data/auth/*`

**优先级：** 🟡 重要　·　凭据解析器：none/api_key(static)、oauth_token(gangtise)、sdk_session(ifind)。

- base：`CredentialResolver`(ABC) + credential_hash(缓存/锁键)
- static：NoneCredentialResolver(恒 ready) / EnvApiKeyResolver(缺 key→missing_config)
- oauth：GangtiseTokenResolver(POST 换 token 按 expiresIn 缓存)；sdk_session：IfindSessionResolver(只校验账号+SDK 路径)
- cache：基于 KV 的 token 缓存读写

**依赖：** finance_data.schemas、httpx、infra.state

### `backend/app/features/finance_memory/service.py`

**优先级：** 🔴 核心　·　金融记忆核心(最大文件)：以 Oryx DB 为权威，records/entities/edges/facts/gaps CRUD + SQL 图上下文 + 抽取审阅。

- 可见性过滤 _visible_record_filter(user-finmemory 仅 owner / shared 需管理员 / graph 只读)
- 图谱物化 _materialize_record_graph(按 kind 生成 EntityFact/Edge，缺口写 Gap，阈值 0.7/0.75 自动建实体)；改删级联软删
- `graph_context`：实体→1-2 跳邻居 edges→facts→evidence，空结果记 gap+agentGuidance
- 抽取流水线 stage/accept/ignore_extraction_batch(候选先入队，接受才落库)；所有变更写 FinanceMemoryEvent 审计

**依赖：** db.models.Finance*、finance.service、lib.clock

### `backend/app/features/finance_portfolio/*`

**优先级：** 🟡 重要　·　纸面(paper)组合：研究专用状态、持仓/交易/快照/再平衡 + 回测。

- router + schemas：PortfolioSummary/Detail/BacktestView 等

**依赖：** db.models.FinancePortfolio*、finance_data.service.run_quote
