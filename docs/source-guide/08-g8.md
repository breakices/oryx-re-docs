# 08 · 前端核心：状态 / API / 两条 SSE 流

> src · types / store / api / hooks

**两条 SSE 流**：turn 级(chatApi.streamTurn 产 Chunk，由 useChat.consumeStream → streamBuffer(content 节流) + streamEvents(其它事件分发到 store) 消费) 与 session 级(backgroundApi.sessionEvents 产 SessionEventChunk，由 useSessionEvents 消费成 bg 任务，并通过 onSpawn 回到 useChat 接管自动派生 turn)。API 层统一经 authFetch(注 Bearer、401 派发事件) 与 sseGenerator(解析 data 帧)；状态收敛在一个 zustand store，按 6 slice 切分。

!!! note "建议阅读顺序"
    main → App → types → store/types → store → slices/chat → 其余 slice → transport → api/types → client → features/chat → features/background → 其余 features → mock → streamBuffer → streamEvents → useChat → useSessionEvents → useAppShellData → 辅助 hooks

### `src/hooks/useChat.ts`

**优先级：** 🔴 核心　·　聊天核心 hook：发消息→chatStart→消费 turn SSE→落库 reload，含 rewind/compact/cancel/resume/spawn 链。

- `consumeStream`：content 走 streamBuffer 节流拼接，其它 chunk 先 flush 再 applyStreamEvent；结束后 reloadVisibleMessages，有 spawn 链则 attachToTurn
- `send`：流式 ref 门禁→乐观插 user+assistant 占位→finance 默认 memorySources→chatStart→真实 id 重命名→消费流
- sendRewind/compact/cancel/resumeTurn；secondary 流用于 inline compact 不抢全局状态

**依赖：** api/client、store、./chat/*
